# UX 방법론 통합 정리 (1~7주차)

---

# 1. UX의 본질

- UX = **사용자의 경험을 설계하는 방법론**
- UI = **그 경험을 구현하는 인터페이스**

### 관계

- UX (상위 개념) > UI (구현 요소)

---

### 디자인의 변화

- 과거: 형태 / 미적 요소 중심
- 현재: **문제 해결 + 경험 설계 + 가치 전달**

---

### 핵심 정의

→ UX = **기존 경험을 새로운 경험으로 바꾸는 것**

---

# 2. UX 방법론의 전체 구조

---

## 2-1. UX = 문제 해결 프로세스

```
문제 → 아이디어 → 시나리오 → 프로토타입 → 솔루션
```

→ UX는 **Solving (문제 해결 과정)**

---

## 2-2. 디자인 씽킹

### 5단계

1. 공감 (Empathize)
2. 정의 (Define)
3. 아이디어 (Ideate)
4. 프로토타입 (Prototype)
5. 테스트 (Test)

### 특징

- 사용자 중심
- 공감 기반 문제 정의
- 반복적 개선 구조

---

## 2-3. 더블 다이아몬드

### 구조

```
Discover → Define → Develop → Deliver
```

- 문제 찾기 → 해결하기

---

### 핵심

- 확산 → 수렴 → 확산 → 수렴

---

# 3. 문제 정의 관점 (UX 사고 방식)

UX는 기능 개선이 아니라

→ **문제를 어떻게 정의하느냐**

---

## 3가지 관점

1. **원칙 중심**
    - 인간 심리 / 인지 기반
2. **프로세스 중심**
    - 사용자 여정 / 데이터
3. **트렌드 중심**
    - 시대 / 시장 흐름

---

## 사고 방식 비교

- 비즈니스: 효율, 최적화
- 디자인: **문제 재정의 + 가치 창출**

---

# 4. UX 설계 구성 요소

---

## UX/UI 5요소 구조

```
IA → User Flow → Interaction → UI → GUI
```

### 설명

1. IA: 정보 구조
2. Flow: 사용자 흐름
3. Interaction: 상호작용
4. UI: 화면 구성
5. GUI: 시각 표현

---

# 5. 사용자 중심 설계 핵심 개념

---

## Affordance

- 디자인이 행동을 유도해야 함

---

## Mental Model

- 사용자가 기대하는 동작 방식
→ 실제 구조와 일치해야 함

---

## Jakob’s Law

- 기존 서비스와 동일한 방식 기대
→ 익숙함 유지 중요

---

# 6. UX 심리 법칙

---

## 기억

- 밀러의 법칙
→ 7±2개

---

## 선택

- 힉스의 법칙
→ 선택 많을수록 느림

---

## 물리적 인터랙션

- 피츠의 법칙
→ 크고 가까울수록 쉬움

---

## 복잡성

- 테슬러의 법칙
→ 복잡성은 제거 불가, 분산 필요

---

# 7. UX 품질 (5주차)

---

## UX 품질 정의

- 사용자 가치 + 비즈니스 목적의 조화

---

## 트레이드 오프

- 충돌하는 가치 중 선택
- 사용자 멘탈모델 기준 판단
- 서비스 정체성 형성

---

## 좋은 UX 조건

- 고충 해결
- 맥락 고려
- 결과 명확
- 적절한 정보 제공
- Good Default
- 트레이드오프 반영

---

## UX 피라미드

### 하위 (필수)

- 유용성
- 신뢰성
- 사용성

### 상위 (가치)

- 편의성
- 감성
- 의미성

→ 아래는 필수, 위로 갈수록 고난이도

---

# 8. UX 구성 요소와 동기 (6주차)

---

## UX 핵심 요소

- 동기 (Motive)
- 행동 (Behavior)
- 맥락 (Context)

---

## UX 흐름 구조

```
동기 → 행동 → 고충 → 니즈 → 해결 → 태도
```

---

## 맥락

- 경험의 배경
- 동기/행동 변화

---

## 빙산 모델

### 수면 위

- 행동, 니즈

### 수면 아래

- 동기, 고충

→ UX 핵심은 **수면 아래**

---

## 동기 유형

### 목적의식적

- 목표 명확
- 단선적 흐름
- 맥락 영향 적음

### 습관적

- 반복적
- 무의식적 행동
- 맥락 영향 보통

---

## 행동

- 경험의 발현
- User Flow 구조

---

## 습관 설계

- 어포던스
- 반복 유도
- 검색 최소화

---

## 고충

- 사용성 문제
- 의미성 문제
- 기능/신뢰 문제

---

## 니즈

- 고충 해결 욕구
- 다양한 해결 가능

---

## 태도

- 서비스 성격 형성

---

## 인간 욕구 (매슬로우)

- 생리 → 안전 → 소속 → 존경 → 자아실현

---

# 9. UX 여정 (7주차)

---

## UX 여정 정의

- 사용자의 전체 경험 과정

---

## 목적

- 고충(Pain Point) 파악
- 기회(Opportunity) 발견
- 경험 흐름 시각화

---

## 특징

- 감정 / 생각 / 행동 / 상호작용 포함
- 모든 접점 고려

---

## UX 여정 단계

```
외부 탐색 → 접근 → 탐색 → 조회 → 주 활동 → 부가 활동 → 평가 → 이탈
```

---

## 단계 설명

- 외부 탐색: 앱, 광고, 알림
- 접근: 진입, 인증, 개인화
- 탐색: 구조 이해, 이동
- 조회: 콘텐츠 소비
- 주 활동: 핵심 기능 수행
- 부가 활동: 선택 행동
- 평가: 경험 판단
- 이탈: 종료 및 재진입

---

## 탐색 vs 조회

- 탐색: 이동 / 구조 이해
- 조회: 콘텐츠 소비

---

## 탐색 유형

1. 홈
2. 메뉴
3. 검색
4. 추천

---

## 사례 (교보문고 바로드림)

- 온라인 → 오프라인 수령
- 고충 발견 → UX 개선

---

# 10. UX 설계의 핵심 목표

- 사용자 고충 해결
- 인지 부하 최소화
- 가치 중심 경험 제공
- 익숙하고 효율적인 인터랙션

---

# 11. 전체 흐름 (최종 정리)

```
사용자 공감 → 문제 정의 → 아이디어 → 설계 → 검증
```

---

# 최종 핵심

```
UX 방법론 =
문제 정의 + 사용자 이해 + 경험 설계 + 검증의 반복 과정
```

→ 목적:

**사용자가 편하게 목표를 달성하도록 새로운 경험을 만드는 것**
